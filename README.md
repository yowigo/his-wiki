**版本**: v1.0.0

## 📚 项目简介

HIS-Wiki 是一个专注于医疗信息系统（Hospital Information System）的知识库，旨在：
- 整理和沉淀HIS系统开发经验
- 提供标准化的开发规范和模板
- 记录业务知识和故障案例
- 建立团队协作的知识共享平台

## 🏗️ 知识库架构

### 三层知识体系
```
his-wiki/
├── schema/                      # 模式定义层（规范、模板、流程）
│   ├── CLAUDE.md                # 小柔助手角色定义
│   └── 规范/                    # 开发规范文档
│       ├── 编码规范.md
│       ├── 接口设计规范.md
│       ├── 数据库设计规范.md
│       └── 公司禁令/           # 开发红线规范
│
├── wiki/                        # 编译知识层（LLM维护）
│   ├── index.md                 # 知识库首页导航
│   ├── log.md                   # 操作日志记录
│   ├── api/                     # 接口开发文档
│   ├── subsystems/              # 业务子系统文档
│   ├── patterns/                # 设计模式文档
│   ├── troubleshooting/         # 故障案例文档
│   └── work-log/                # 工作日志文档
│
└── raw/                         # 原始资料层
    └── README.md                # 原始资料说明
```

## 🎯 核心内容

### 1. 开发规范
- **编码规范**: 统一代码风格和质量标准
- **接口设计规范**: RESTful API设计原则
- **数据库设计规范**: Schema划分、命名规范、SQL规范
- **公司禁令**: 编码、实施、产品管理红线

### 2. 业务知识
- **HIS业务概览**: 系统定位、项目关系、核心业务
- **门诊流程**: 挂号→接诊→开医嘱→收费→执行→报告
- **住院流程**: 入院登记→医嘱→执行→费用→出院结算
- **费用结算**: 医保结算、支付处理、票据管理
- **检验检查**: 检验流程、互认平台、影像管理
- **药品管理**: 药品医嘱、发药流程、库存管理
- **患者管理**: 患者建档、身份识别、就诊管理

### 3. 接口开发模板
- **插件开发模板**: .NET Framework接口组插件开发
- **CRUD接口模板**: .NET 8.0 HIS 5.1标准CRUD接口
- **查询接口模板**: 分页查询、复杂筛选、关联查询
- **第三方对接模板**: 医保平台、互认平台、药事平台对接
- **医保插件模板**: 各地医保平台对接开发

### 4. 支持文档
- **设计模式**: 医嘱状态机、GitHub备份经验等
- **故障案例**: 实际项目中的问题和解决方案
- **工作日志**: 项目进展和总结记录

## 🚀 快速开始

### 1. 克隆仓库
```bash
git clone https://github.com/yowigo/his-wiki.git
cd his-wiki
```

### 2. 浏览知识库
- 查看知识库首页: `wiki/index.md`
- 查找开发规范: `schema/规范/`
- 学习业务知识: `wiki/subsystems/`
- 使用接口模板: `wiki/api/`

### 3. 使用建议
- **新员工**: 从`wiki/subsystems/overview.md`开始了解HIS系统
- **开发人员**: 参考`schema/规范/`和`wiki/api/`进行开发
- **实施人员**: 学习`wiki/subsystems/`中的业务流程
- **管理人员**: 查看`wiki/work-log/`了解项目进展
