## 数据库环境

- **数据库**：PostgreSQL 12+
- **访问方式**：UPTools 跨域访问（接口组）/ Entity Framework Core（HIS 5.1）

## Schema 组织

HIS 系统按业务域划分 Schema：

| Schema | 说明 | 典型表 |
|--------|------|--------|
| public | 通用业务数据 | lab_apply、lab_collect、lab_sample |
| qw_base | 基础配置数据 | b_org、b_clin_item、b_lab_type |
| fee | 费用数据 | fee_detail |
| insur | 医保/互认数据 | hr_report_info、hr_upload_record |
| outpatient | 门诊数据 | pt_diagnosis、outp_fee_detail |
| inp | 住院数据 | pt_diagnosis、inp_fee_detail |
| drug | 药品数据 | — |
| lab | 检验数据 | — |
| pacs | 检查/影像数据 | — |

## SQL 规范

### 必须使用参数化查询

```csharp
// 正确
var sql = "SELECT * FROM public.lab_apply WHERE sample_id = @sampleId";
var result = UPTools.Query(sql, new { sampleId = id });

// 错误（禁止）
var sql = $"SELECT * FROM public.lab_apply WHERE sample_id = '{id}'";
```

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写下划线 | `lab_report_indicator` |
| 字段名 | 小写下划线 | `patient_id` |
| 主键 | id 或 表名_id | `report_id` |
| 外键 | 关联表_id | `patient_id` |
| 时间字段 | xxx_time 或 xxx_date | `create_time` |
| 状态字段 | is_xxx 或 status | `is_uploaded` |
| 索引 | idx_表名_字段名 | `idx_lab_apply_sample_id` |

### 查询优化

- 大数据量查询必须分页
- 合理使用索引
- 避免 SELECT *，明确列出所需字段
- 复杂查询添加注释说明业务含义
