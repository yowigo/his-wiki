### 日志规范

使用 log4net，通过 `LogTools` 工具类调用：

```csharp
// 信息日志
LogTools.Info("开始获取插件信息");

// 错误日志（含客户端ID）
LogTools.Error($"[客户端ID：{clientId}] 执行方法{methodName}时出错");
```

配置文件：`config/cpapilog.config`  
日志目录：`cpapilog/`，默认保留 60 天

**关键节点必须写信息日志**，方便排查问题：

```csharp
// 接口请求前：记录入参
LogTools.Info($"[{action}] Request: {jsonParams}");

// 接口响应后：记录返回内容
LogTools.Info($"[{action}] Response: {result}");

// 关键业务节点：记录中间状态
LogTools.Info($"[{action}] PatientId={patientId}, InsureNo={insureNo}, Step=ReadCard");
```

### 异常处理

```csharp
try
{
    var result = helper.ProcessRequest(action, jsonParams);
    return result;
}
catch (Exception ex)
{
    LogTools.Error($"[{action}] Failed: {ex.Message}", ex);
    return new PluginResult
    {
        Code = -1,
        Message = $"执行 {action} 失败：{ex.Message}"
    };
}
```

## 运行时规范

**禁止数据映射 fallback：**
- 数据映射查不到时，必须明确报错 / 日志告警 / 跳过，绝不用原始值静默回退

**Blazor JS 规则：**
- 组件内禁止内联 `<script>`（Blazor 会屏蔽组件级 script 执行）
- 所有 JS 放 `wwwroot/js/`，在 `App.razor` 通过 `<script src="...">` 引入

**MutationObserver 正确写法：**
- 回调里必须检测 `addedNodes`，不能用 `getElementById` 判断元素存在（会触发无限循环）

```js
const _observer = new MutationObserver((mutations) => {
    for (const m of mutations) {
        for (const node of m.addedNodes) {
            if (node.nodeType !== 1) continue;
            if (node.id === 'target' || node.querySelector?.('#target')) {
                doWork();
                return;
            }
        }
    }
});
_observer.observe(document.documentElement, { childList: true, subtree: true });
```

**DataGridView 列命名：**
- 手动创建列时必须同时设 `Name` 和 `DataPropertyName`
- `Cells["xxx"]` 按名称访问依赖的是 `Name`，不是 `DataPropertyName`
