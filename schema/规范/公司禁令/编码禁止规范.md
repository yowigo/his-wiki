# 产品编码禁止规范 V2.5

> 来源：Miro 公司规范文件（2026-01-01 生效）
> 适用人员：产品开发人员

---

## 第一章 制定目的

为了产品无重大缺陷，根据历史事件案例总结而制定编码禁止规范。内部各小组转达、传达编码禁止规范要求。

## 第二章 适用范围和人员

产品开发人员适用。

## 第三章 禁止写法、做法

### 1. 禁止使用无 where 删除

错误示例：

```csharp
// 直接删除数据表中所有记录，导致数据被清空
db.Delete(table).Execute();
// 直接执行 SQL 语句
db.Sql("Delete From table;").Execute();
```

正确示例：

```csharp
// 需要增加删除条件防止数据被清空
db.Delete(table).Where(...).Execute();
db.Sql("Delete From table Where 列=条件").Execute();
```

### 2. 禁止使用无 where 更新

错误示例：

```csharp
// 以下方式将导致整体数据表的行更新成相同的值的灾难
db.Update(Table)
  .Column(列1, 值)
  .Column(列2, 值)
  .Execute();
```

正确示例：

```csharp
// 需要带有指定的更新条件，防止整表更新
db.Update(Table)
  .Column(列1, 值)
  .Column(列2, 值)
  .Where(条件列, 值)
  .Execute();
```

### 3. 禁止使用无 where 查询（大表，千行以上）

错误示例：

```csharp
// 使用无条件查询导致查询效率慢，内存占用高，数据连接资源不能快速释放
db.Select(table).Column("*").GetModelList();
```

正确示例：

```csharp
db.Select(table)
  .Column("列")
  .Where(条件, 值)
  .GetModel();
```

### 4. 禁止出现死循环或无限递归调用

```csharp
While(条件)
{
    if(xxx) {
        条件 = false;
    } // 需要防止条件无法进入，循环不退出
} // 循环尽量避免循环中开启关闭数据库
```

### 5. 禁止不关闭文件流、内存流

错误示例：

```csharp
{
    var stream = File.Read("c:\\aa.file");
    var data = new byte[stream.Length];
    stream.ReadByte(data);
} // stream 并没有释放，一直在内存中，待垃圾回收处理
```

正确示例：

```csharp
{
    using(var stream = File.Read("c:\\aa.file"))
    {
        var data = new byte[stream.Length];
        stream.ReadByte(data);
    } // using 方式会自动释放
    // 也可以通过 stream.Close() 显式关闭流
}
```

### 6. 禁止不释放数据库连接

错误示例：

```csharp
{
    // 声明 Db 连接对象，并没有释放连接
    var db = new DbContext();
    var cnt = db.Insert<T>(model).Execute();
}
```

正确示例：

```csharp
{
    using(var db = new DbContext())
    {
        var cnt = db.Insert<T>(model).Execute();
    } // using 时会自动释放连接资源
}
```

### 7. 禁止错误使用事务

- 禁止对数据库的增删改不开启事务
- 禁止使用了事务，不提交事务，也不回滚事务

错误示例：

```csharp
{
    var db = new DbContext(true);
    var cnt = db.Insert<T>(model).Execute();
    cnt = db.Update<T>(model).Where("id", 123).Execute();
} // 没有提交事务，也没有释放资源，导致该行记录一直被占用，其它用户操作将长期阻塞
```

正确示例：

```csharp
{
    using(var db = new DbContext(true))
    {
        var cnt = db.Insert<T>(model).Execute();
        if(cnt == 0) db.Rollback();

        cnt = db.Update<T>(model).Where("id", 123).Execute();
        if(cnt == 0) db.Rollback();

        db.Commit();
    } // 释放连接资源
}
```

### 8. 禁止重复开启事务

错误示例：

```csharp
{
    using(var db = new DbContext(true))
    {
        // 参数 true 已自动启用事务，手动再开启会导致连接池溢出
        db.BeginTrans();
        ...
        db.CommitTrans();
    }
}
```

正确示例：

```csharp
// 方式一：自动开启
using(var db = new DbContext(true))
{
    ...
    db.CommitTrans();
}

// 方式二：手动开启
using(var db = new DbContext())
{
    db.BeginTrans();
    ...
    db.CommitTrans();
}
```

### 9. 禁止无限制开启多线程

错误示例：

```csharp
{
    for(var i = 0; i < list.Count; i++)
    {
        var thread = new Thread(..);
        thread.Start();
    } // 线程开启不受控制，Web 并发时 CPU 资源将被快速耗尽
}
```

正确做法：共同评估是否需要多线程，单线程是否无法解决问题。

### 10. 禁止使用某个列或几个列时，查询所有列

错误示例：

```csharp
// select * from table 会查出所有列，包括大对象字段
var models = db.Select(Table)
    .Column("*")
    .Where("id", 123)
    .GetModelList();
```

正确示例：

```csharp
var models = db.Select(Table)
    .Column("id")
    .Column("name")
    .Column("state")
    .Where("id", 123)
    .GetModelList();
```

### 11. 禁止查询整表数据到内存中进行分页或过滤

错误示例：

```csharp
// 查询整表到内存中再过滤，导致内存增长、数据库读取开销增加
var list = db.Sql("select * from table").GetModelList();
list.Find(obj => obj.id == "123");
```

正确示例：

```csharp
var list = db.Sql("select 列1,列2,列3... from table where id = @id")
    .Parameters("id", 123)
    .Paging(1, 20)
    .GetModelList();
```

### 12. 禁止使用 SQL 拼装字符串

错误示例：

```csharp
// SQL 拼接缺点：容易被注入、容易拼错、数据库不会缓存
var sql = $"select id,name from table a where id='{id}'";
db.Sql(sql).GetModelList();
```

正确示例：

```csharp
var sql = "select id,name from table a where id=@id";
db.Sql(sql)
    .Parameter("id", 123)
    .GetModelList();
```

### 13. 禁止 SQL 条件使用 IN（IN 100 个条件以上）

错误示例：

```sql
select id, pt_name from pt_info where vists_id in (select id from pt_vists)
```

正确示例：

```sql
select id, pt_name from pt_info p, pt_vists v where p.vists_id = v.id
```

### 14. 禁止业务代码 try...catch 中 catch 不做任何处理

错误示例：

```csharp
try
{
    // 业务代码
}
catch
{
    // 没有任何记录
}
```

正确示例：

```csharp
try
{
    // 业务代码
}
catch(Exception ex)
{
    Log($"调用模块:xxx,错误描述:{ex.Message}", ex);
    return false;
}
```

### 15. 禁止可能为空的对象不进行判断，直接使用

错误示例：

```csharp
var obj = GetObj("123");
obj.xx; // 如果返回值为空，直接报错：未将对象引用到对象实例
```

正确示例：

```csharp
var obj = GetObj("123");
if(obj != null)
{
    obj.xx;
}
```

### 16. 禁止使用 async 时返回类型为 void

错误示例：

```csharp
// 内部异常会导致进程崩溃，特别是 Web 服务端应用
public async void XXX()
{
    ...
}
```

正确示例：

```csharp
public async Task XXX()
{
    ...
}
```

Foreach 中用异步等效于 async void，错误示例：

```csharp
list.Foreach(async (i) => {
    throw new Exception("引发异常");
});
```

正确做法：

```csharp
var tasks = list.Select(i => Task.Run(async () => {
    throw new Exception("引发异常");
})).ToList();

Task.WaitAll(tasks.ToArray());
```

### 17. 禁止随意使用 Cookie

Cookie 在所有请求中都会带上，非全局应用禁止使用。如果必须使用则必须设置过期时间。

### 18. 禁止错误使用日志级别

禁止将 INFO 类型消息写成 ERROR 类型。日志级别由低到高：

| 级别 | 用途 |
|------|------|
| TRACE | 详细调试信息，追踪内部执行流程 |
| DEBUG | 调试信息，帮助查找定位问题 |
| INFO | 一般运行信息，关键操作成功/失败 |
| WARN | 潜在问题或异常，不影响正常运行 |
| ERROR | 无法恢复的错误，可能导致中断 |
| FATAL | 严重错误，可能导致应用崩溃 |

使用场景：
- **开发阶段**：TRACE 或 DEBUG
- **测试阶段**：INFO
- **生产环境**：WARN、ERROR、FATAL

### 19. 禁止不加密传输信息

调用 WebAPI 时需加密输入参数与返回参数，包括不限于：姓名、电话、密码、家庭地址、身份证号、联系地址等隐私信息和重要业务信息。

### 20. 禁止不走软件开发测试流程，私自加入任何内容到产品包

新开发或修复的程序必须走开发、自测、功能与性能测试、回归测试的发版流程，不允许跳过流程直接打进产品包。工具类软件需执行两款以上杀毒软件查杀。

### 21. 禁止前端不加版本号

js 文件不加版本号导致客户端缓存影响更新后的功能。

### 22. 禁止 js 动态创建 iframe 时不加 id 属性，src 地址后不添加时间戳

添加 id 避免重复加载，src 后添加时间戳防止浏览器缓存。

### 23. 禁止 js 使用资源后不释放

```javascript
// 定时器：使用后必须清除
const intervalId = setInterval(() => {
    console.log('This will run every second');
}, 1000);

if (intervalId) {
    clearInterval(intervalId);
}

// 事件监听器：组件卸载时必须移除
const handleClick = () => { console.log('Window clicked'); };
window.addEventListener('click', handleClick);
window.removeEventListener('click', handleClick);
```

### 24. 禁止一次能完成的数据库操作采用循环多次操作

禁止为了方便写代码，直接采用循环多次访问数据库。

### 25. 禁止开放接口未授权匿名访问

所有接口必须采用 token 身份认证授权才能访问，除登录或采用其他授权方式的接口。

### 26. 禁止未授权越权访问

禁止将接口请求中的 Authorization 替换为低权限用户后仍能访问非授权功能。例如普通医生登录后，直接用 token 访问院长的业务功能接口，导致涉密数据泄露。

### 27. 禁止不预防 js 跨站脚本攻击

前端文本框、文本域、富文本框等必须进行输入验证、输出编码，过滤非法字符，防止特殊字符被解释为 HTML 或 JavaScript 代码。

### 28. 禁止产品账号登录不启用强密码规则

支持参数配置启用强密码规则、密码有效期。启用后密码长度 8 位以上，必须包含数字、字母、特殊字符。到期前 7 天提示修改，到期后必须修改才能登录，修改密码不能与原密码相同。

### 29. 禁止多次登录失败不锁定账户

支持参数配置启用登录失败锁定账户、失败次数、锁定时长（分钟）、无操作后锁屏时间。

### 30. 禁止身份鉴别信息未注销

退出登录时，服务端 token（redis 等）、客户端（cookie、localStorage、sessionStorage 等）等登录认证信息必须删除处理。

### 31. 禁止登录失败时精准提示

登录验证时无论用户不存在还是密码错误，统一提示"用户名或密码错误"，严禁提示"密码错误"。

### 32. 禁止任意文件上传

文件上传前后端需验证指定文件类型才允许上传，允许类型：`.bmp`、`.jpeg`、`.jpg`、`.png`、`.gif`、`.dcm`、`.pdf`、`.zip`、`.mp4`、`.avi`、`.doc`、`.docx`、`.xls`、`.xlsx`、`.dbf`、`.csv`、`.txt`

### 33. 禁止文件上传不控制文件大小

必须控制文件大小，如用户头像、签名信息等控制在 1MB 以内。

### 34. 禁止使用触发器

高并发时触发器会导致额外性能问题，增加排查复杂性，设计不当容易出现数据不一致。生产环境禁止使用触发器。

### 35. 禁止前端不做重复提交验证

保存、发送、审核、确定等操作发送请求后，必须禁用按钮防止用户重复点击产生重复数据。

### 36. 禁止在数据库事务中调用 WebAPI

事务中调用 WebAPI 会导致事务时间过长，API 超时或阻塞时该行被锁定，其它请求出现大量队列等待，严重影响数据库性能。禁止在事务中调用 WebAPI、处理文件、执行休眠等与数据库无关的操作。
