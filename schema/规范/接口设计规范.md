## 接口组核心架构：插件化

接口组所有业务功能通过**插件系统**实现，继承自 `CPAPIPluginBase` 抽象基类。

### 请求处理流程

```
前端操作 → RouterCore 路由分发 → MessageBus 消息总线
    → PluginHandle 反射加载插件 → ProcessRequest(action, jsonParams)
    → action 参数映射到 Helper 类具体方法
    → 返回 PluginResult → 更新 UI
```

### 插件基类

```csharp
public class MyPlugin : CPAPIPluginBase
{
    public override string PluginName => "插件中文名称";
    public override string Description => "插件功能描述";

    public override PluginResult ProcessRequest(string action, string jsonParams)
    {
        // action 参数指定要执行的方法名
        // jsonParams 为 JSON 格式的请求参数
    }
}
```

### PluginResult 标准返回

```csharp
public class PluginResult
{
    public int Code { get; set; }      // 0=成功，非0=失败
    public string Message { get; set; } // 提示信息（中文）
    public object Data { get; set; }    // 业务数据
}
```

## 接口分类

| 类型 | 说明 | 典型场景 |
|------|------|---------|
| 数据同步 | 系统间数据上传/下载 | 检验结果上传、互认报告下载 |
| 查询接口 | 数据检索 | 患者查询、报告列表查询 |
| 业务操作 | 执行业务动作 | 医保结算、对码 |
| 第三方对接 | 外部平台集成 | 医保平台、检验互认平台 |
| 设备接口 | 外设交互 | 读卡器、POS机、体检设备 |

## 接口触发机制

插件通过 HIS 模块的钩子函数触发：

| 触发方式 | 模块 | 钩子函数 | 说明 |
|----------|------|---------|------|
| 事件触发 | outp_doc_wksta | after_outp_recvt | 门诊接诊后 |
| 事件触发 | inp_doctor_wksta | before_doctor_next_patient | 住院切换患者前 |
| 事件触发 | lab_wksta | after_lab_audit | 检验审核后 |
| 保存前校验 | outp_doc_wksta | before_lab_apply_save | 开检验医嘱前 |
| 保存前校验 | inp_doctor_wksta | before_inp_order_save | 保存住院医嘱前 |

## 第三方平台对接模式

### 认证方式

| 平台 | 认证方式 | 数据格式 |
|------|---------|---------|
| 广东省互认 | Token认证（SM2公钥，6h缓存） | XML |
| 北京CHIMA | SM4加密 + SM3签名 | JSON |
| 医保平台 | 各地标准不同 | 各地标准不同 |

### 通用对接流程

```
1. 读取配置文件获取平台信息（端点、凭据）
2. 认证/获取Token
3. 组装请求数据（从HIS数据库查询）
4. 加密/签名（如需要）
5. 发送HTTP请求
6. 解密/解析响应
7. 写入本地数据库记录
8. 返回PluginResult
```

## 配置管理

每个插件有独立配置文件，通过 `Common.GetFromConfig(name)` 读取：

```
插件目录/
├── MyPlugin.exe
├── MyPlugin.config          # 插件配置（XML格式）
└── version.json             # 版本信息
```

### version.json 结构

```json
{
    "currentVersion": "1.0.0",
    "hisClientVersion": "5.0.0",
    "flieList": ["MyPlugin.exe", "MyPlugin.dll"],
    "versionDescribe": "版本说明"
}
```

## 数据库访问

- 使用 UPTools 进行跨域数据库访问
- 优先使用参数化查询
- 大数据量查询必须分页

## 日志规范

接口入口和出口必须记录日志：

```csharp
LogTools.Info($"[{action}] Request: {jsonParams}");
// ... 业务逻辑
LogTools.Info($"[{action}] Response: code={result.Code}");
```

异常必须记录完整堆栈：

```csharp
LogTools.Error($"[{action}] Failed: {ex.Message}", ex);
```
