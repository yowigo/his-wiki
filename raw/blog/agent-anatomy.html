§ 02
行动之环 / The Agent Loop
Fig. 02 — Cyclic

循环,不是 while, 而是 自我询问 的 while。

普通程序的 while 靠一个布尔表达式判断退出;Agent 的 while 把"是否该停"交给被循环者自己。每一步,LLM 读完上下文,吐出下一步想做什么——如果是调用工具,Harness 负责执行并把结果喂回;如果是"完成",循环就结束。

上下文管理器是第三个角色。它决定哪些历史被保留、哪些被摘要、哪些被丢弃——因为 token 窗口是有限的。

while not done:
  thought = LLM(context)
  if is_tool_call(thought):
    result = execute(thought)
    context += result
  else:
    done = True

§ 04
逐层展开 / Layer by Layer
Fig. 04 — Index
Depth01
Harness 马具
Harness = 循环控制器 + 工具接口 + 上下文管理器 差异点:和普通 while 不同,这里的"退出条件"不是布尔值,而是由被循环者(LLM)自己判断任务是否完成。
Depth02
LLM 语言模型
LLM = 神经网络 + 自回归生成 + 训练出的分布 差异点:相比"一次性输出答案"的分类网络,LLM 是串行吐字的;相比规则系统,它的行为来自训练出的概率分布,而非手写规则。
Depth03
Loop Controller 循环控制器
Controller = while 循环 + 终止判断 + 步数上限 差异点:条件本身由被循环的对象自己决定。必须配一个硬性步数上限兜底,防止无限循环。
Depth04
Tool Interface 工具接口
Tool = 函数签名 + 文本解析器 + 错误回灌 差异点:普通 API 调用方是代码,参数是类型化的;工具接口的调用方是自然语言文本,多出"解析→校验→错误反馈"一环。
Depth05
Context Manager 上下文管理器
Context = 消息队列 + 窗口裁剪 + 记忆摘要 差异点:普通队列先进先出,但系统提示、最近几步、关键中间结果必须保留,因此需要摘要策略把久远对话压缩再塞回。
Depth06
Autoregression 自回归生成
Autoreg = 概率采样 + token 拼接 + 每步重算 差异点:每抽一个 token,整个输入序列就变了,要再跑一遍网络。这个"抽-拼-再算"循环直到 <eos>。
Depth07
Attention 注意力机制
Attention = 点积相似度 + softmax 归一化 + 加权求和 (Q/K/V) 差异点:同一个 token 通过三个不同线性变换同时扮演"提问者、被查者、信息载体"三种角色,这是 Transformer 超越 RNN 的根本原因。
§ 05
不可再分的原语 / Irreducible Primitives
Fig. 05 — Atoms
⊗
Primitive · 01
Matrix Multiply
矩阵乘法——全部变换的载体。从线性层到注意力权重,都是它。
Σ
Primitive · 02
Softmax
把任意实数向量变成"加起来等于 1"的概率分布。让选择变得可比。
+
Primitive · 03
Addition
加法——残差连接让梯度能流过百层深网,没有它就没有今天的大模型。
∫
Primitive · 04
Non-linearity
ReLU / GELU——打破线性,让网络有能力拟合任意函数。
§ 06
把树长出去 / Extend the Tree
Fig. 06 — Leverage
Case · 01
Multi-Agent
Multi-Agent =
Agent + 通信协议
+ 角色分工
任务分解并行化,让多个 Agent 协作。只是在老树上多接了两个节点。
Case · 02
RAG
RAG =
LLM + 向量检索
+ 上下文注入
让 LLM 拥有"查资料"能力。注意力的外延 —— 把外部文档拉进窗口。
Case · 03
MCP
MCP =
工具接口 + 标准协议
+ 跨进程通信
把"工具接口"变成跨进程标准,让任何程序都能成为 Agent 的手脚。
