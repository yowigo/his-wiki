图5：Business.ClincSettle() 门诊结算编排流程

flowchart TD
    START([Business.ClincSettle 开始]) --> INIT[InsureBusiness.InitInsure
初始化医保接口]
    INIT --> INIT_OK{初始化成功?}
    INIT_OK -->|失败| ERR1[返回 code:0
初始化失败]
    INIT_OK -->|成功| SPLIT{is_split == 1?}

    SPLIT -->|是 拆分模式| SWAP[InsureBusiness.ClinicSwap
直接正式结算]
    SPLIT -->|否 标准模式| PRE[InsureBusiness.ClinicPreSwap
预结算]

    PRE --> PRE_OK{预结算成功?}
    PRE_OK -->|失败| ERR2[返回 code:0
预结算失败]
    PRE_OK -->|成功| PRE_SPLIT{返回 is_split == 1?}

    PRE_SPLIT -->|是| BAL[InsureBusiness.SelfBalance
获取账户余额]
    PRE_SPLIT -->|否| SWAP

    SWAP --> SWAP_OK{结算成功?}
    SWAP_OK -->|失败| ERR3[返回 code:0
结算失败]
    SWAP_OK -->|成功| BAL

    BAL --> BAL_OK{余额查询成功?}
    BAL_OK -->|成功| RET([组织返回值
code:1 + data])
    BAL_OK -->|失败| DEL[InsureBusiness.ClinicDelSwap
★ 自动撤销结算]
    DEL --> ERR4[返回 code:0
余额查询失败]

图6：住院结算超时冲正机制（SettleSwap）

flowchart TD
    START([SettleSwap 开始]) --> QUERY[查询档案 + 就诊记录 + 预结记录]
    QUERY --> BUILD[构建 2304 请求
psn_no, mdtrt_id, medfee_sumamt...]
    BUILD --> CALL["🔗 HttpHelper.YBBusiness('2304')
住院结算"]

    CALL --> OK{请求成功?}
    OK -->|成功| CHECK_ERR{医保误差 > 0.1元?}

    OK -->|失败| TIMEOUT{错误含 'timed out'?}
    TIMEOUT -->|否| CANCEL[撤销出院登记
返回失败]
    TIMEOUT -->|是| FLUSH["🔗 HttpHelper.YBBusiness('2601')
冲正交易"]

    FLUSH --> FLUSH_OK{冲正成功?}
    FLUSH_OK -->|失败| FAIL1[返回 code:0
冲正失败]
    FLUSH_OK -->|成功| ASK{弹窗: 是否重试结算?}
    ASK -->|是| CALL
    ASK -->|否| CANCEL

    CHECK_ERR -->|否| SAVE[INSERT insur.ins_balance
保存结算记录]
    CHECK_ERR -->|是| FLUSH2["🔗 冲正交易 2601
误差过大自动冲正"]
    FLUSH2 --> FAIL2[返回 code:0
医保误差超限]

    SAVE --> RET([返回 code:1
结算成功])
