# 文档说明

## 概要描述

本文档是飞跃医疗信息系统与第三方应用之间数据交互服务开发的标准说明。

## 阅读对象

开发、实施成员、涉及使用飞跃医疗信息系统与三方对接的人员。

# 数据交互技术规范

目前发布的Webapi服务为JSON报文格式

## HTTP通讯说明（JSON格式）

通过HTTP+POST方式访问，请求数据通过JSON方式传递

内容类型：application/json; charset=utf-8

请求地址(案例)： <http://192.168.21.41:13100/>对应服务地址

### 请求数据说明:

{

system\_code:"S0001-自助机，S0002-公众号,S0099-其他",

system\_name:"渝北人民医院自助机 ",

各接口参数

}

### 返回数据说明

{
    **"code"**:**200**,
    **"msg"**:**"查询成功"**,
    **"data"**:[]

}

# 业务流程

## 基础数据

### 获取诊疗项目

1. 获取HIS诊疗项目基础数据到第三方（配置体检套餐）。

![](data:image/png;base64...)

## 体检业务

### 体检个人登记

1. 单个体检病人登记，流程包含：患者建档、体检项目登记、费用划价、费用状态回写、检验\检查报告获取。
2. B2003和B2003A任选其一， B2003A档案不存在，直接新增返回，然后进行登记业务。
3. 收费划价单生成成功后才能写入检验、检查申请单，同步调用。
