## 基本流程

```
挂号 → 接诊 → 开医嘱 → 发送 → 收费 → 执行 → 报告
```

## 接口组触发点

| 触发时机 | 钩子函数 | 工作站 | 接口动作 |
|----------|---------|--------|---------|
| 医生接诊后 | after_outp_recvt | outp_doc_wksta | 上传患者信息、查询互认报告 |
| 开检验医嘱前 | before_lab_apply_save | outp_doc_wksta | 精准互认匹配 |

## 相关数据表

- `outpatient.pt_diagnosis` — 门诊诊断
- `outpatient.outp_fee_detail` — 门诊费用明细

## 详细流程说明

### 1. 挂号阶段
- 患者信息登记
- 科室选择
- 医生排班匹配
- 挂号费用计算

### 2. 接诊阶段
- 医生工作站调用
- 患者信息加载
- 历史病历查看
- 接口触发：`after_outp_recvt`

### 3. 开医嘱阶段
- 检验医嘱开具
- 检查医嘱开具  
- 药品医嘱开具
- 接口触发：`before_lab_apply_save`

### 4. 收费阶段
- 费用计算
- 医保结算
- 自费部分收取
- 票据打印

### 5. 执行阶段
- 检验科执行
- 影像科执行
- 药房发药
- 治疗执行

### 6. 报告阶段
- 检验报告生成
- 检查报告生成
- 医生查看报告
- 患者获取报告

## 接口集成点

### 医保接口
- 实时结算
- 费用明细上传
- 结算结果返回

### 互认平台接口
- 患者信息上传
- 互认报告查询
- 检验结果上传

### 支付接口
- 银联POS对接
- 移动支付对接
- 电子票据生成

## 数据流向

```
患者 → 挂号系统 → 医生工作站 → 医嘱系统 → 收费系统
      ↓           ↓           ↓           ↓
接口组 → 医保平台 → 互认平台 → 支付平台 → 报告系统
```
