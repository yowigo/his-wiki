## 检验流程

```
医生开检验医嘱 → 采集标本 → 送检 → 检验 → 审核 → 出报告
```

## 接口组触发点

| 触发时机 | 钩子函数 | 工作站 | 接口动作 |
|----------|---------|--------|---------|
| 技师审核后 | after_lab_audit | lab_wksta | 上传检验数据到互认平台 |
| 放射审核后 | after_rad_rpt_audit | rad_wksta | 上传影像数据到互认平台 |

## 检验互认

检验检查互认是接口组的重要业务，支持：
- **广东省** — 省级检验检查互认
- **北京CHIMA** — 北京标准化互认平台

互认插件：`plugins.eb_testrecongnition`

### 互认流程

```
医生接诊 → 查询外院互认报告 → 展示报告列表
    → 医生决策（互认/不互认）→ 记录决策原因
    → 开医嘱时精准匹配已有报告 → 提醒医生避免重复检查
    → 结果回传互认平台
```

## 详细业务流程

### 1. 检验申请阶段
- 医生开具检验医嘱
- 检验项目选择
- 标本类型指定
- 紧急程度标识

### 2. 标本采集阶段
- 患者准备
- 标本采集
- 标本标识
- 送检登记

### 3. 检验执行阶段
- 标本接收
- 检验项目分配
- 仪器检测
- 结果录入

### 4. 报告审核阶段
- 技师审核
- 医生审核
- 报告发布
- 接口触发：`after_lab_audit`

### 5. 报告查看阶段
- 医生工作站查看
- 患者自助查询
- 报告打印
- 电子报告发送

## 检查（影像）流程

### 基本流程
```
医生开检查申请 → 预约安排 → 检查执行 → 影像采集 → 报告撰写 → 报告审核 → 报告发布
```
