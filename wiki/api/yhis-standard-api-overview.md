# 飞跃智慧系统标准化接口文档（HIS 接口 1.0）

> 原始资料：[raw/HIS接口文档1.0/](../../raw/HIS接口文档1.0/)
>
> 本文档汇总了飞跃智慧系统对外提供的标准化 REST 接口，供第三方系统（区卫平台、互联网医院、PACS、LIS、手麻系统等）对接使用。

---

## 接口规范

### 通信协议

| 项目 | 说明 |
|------|------|
| 协议 | HTTP/HTTPS |
| 数据格式 | JSON |
| 编码 | UTF-8 |
| 统一响应 | `{ "code": 200, "msg": "", "data": {}, "timestamp": 1690165170 }` |

### 响应码

| code | 含义 |
|------|------|
| `200` | 成功 |
| `500` | 失败 |

### 服务地址前缀

```
instance/publish/yhis/V1/
instance/publish/yhis/V1.1/   ← 部分接口有 V1.1 版本
```

## 接口编码规则

| 前缀 | 所属模块 |
|------|----------|
| `A4*` | 药品、卫材、物资、设备（进销存） |

## 入参通用字段

几乎所有接口都需要传入 `org_id`（机构ID）作为基础参数。部分接口支持 `key_word` 关键字查询。

### 常见入参模式

```json
{
  "org_id": "a5c69f5832c99b31",    // 机构ID，必填
  "dept_id": "xxx",                 // 科室ID
  "pt_id": "xxx",                   // 患者ID
  "key_word": "张三"                // 关键字查询
}
```

### 常见出参模式

```json
{
  "code": 200,
  "msg": "成功",
  "data": [
    { /* 业务对象 */ }
  ],
  "timestamp": 1690165170,
  "innermsg": ""
}
```

## 版本管理

部分接口存在多版本（V1.0 / V1.1），主要差异：

| 版本 | 特点 |
|------|------|
| V1.0 | 基础字段返回 |
| V1.1 | 扩展字段返回（如科室简介、电话、地址等） |

调用方式：通过服务地址路径区分
```
V1.0:  instance/publish/yhis/V1/GetWosDeptByOrgId
V1.1:  instance/publish/yhis/V1.1/GetWosDeptByOrgId
```
