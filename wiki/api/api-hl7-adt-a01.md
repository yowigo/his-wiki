# ADT^A01 入院通知接口 - 患者入院登记

## 基本信息
- **接口类型**: HL7 v2.5
- **消息类型**: ADT^A01 (患者入院)
- **触发条件**: 患者办理入院手续时
- **响应要求**: 同步响应，超时时间30秒
- **消息方向**: HIS → 各子系统（LIS、PACS、药房等）

## 消息结构

### 请求消息（HIS发送）
```
MSH|^~\&|HIS|BEIJING_HOSPITAL|LIS|LAB_DEPT|20250710143000||ADT^A01|MSG202507101430001|P|2.5
EVN|A01|20250710143000

PV1|1|I|301^3楼^01床^^BEIJING_HOSPITAL||||||||||||||||||||||||||||||||||||||20250710
```

## 业务规则

### 触发条件
1. **入院登记完成**: 患者在住院处完成所有入院手续
2. **床位分配确认**: 系统已为患者分配具体床位
3. **费用预交完成**: 患者已缴纳住院押金

### 处理流程
```
患者入院 → 住院处登记 → 床位分配 → 费用预交 → 触发ADT^A01 → 通知各子系统
```

### 接收系统要求
1. **LIS系统**: 创建患者检验档案，准备接收检验医嘱
2. **PACS系统**: 创建患者影像档案，准备接收检查申请
3. **药房子系统**: 创建患者药历，准备接收药品医嘱
4. **护士站系统**: 更新床位状态，准备接收护理医嘱

### 时效性要求
1. **消息发送**: 入院登记完成后5分钟内必须发送
2. **响应超时**: 接收系统必须在30秒内返回ACK
3. **重试机制**: 如果超时未收到响应，每5分钟重试一次，最多重试3次

## 错误处理

### ACK响应代码
| 代码 | 含义 | 处理建议 |
|------|------|----------|
| AA | 应用接受 | 正常处理，无需干预 |
| AE | 应用错误 | 检查应用日志，联系系统管理员 |
| AR | 应用拒绝 | 验证消息格式和权限配置 |
| CA | 提交接受 | 异步处理中，等待后续确认 |

### 常见错误场景
| 错误场景 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 消息解析失败 | HL7格式错误 | 检查字段分隔符和转义字符 |
| 患者ID不存在 | 患者未在接收系统注册 | 先在接收系统创建患者基础信息 |
| 床位信息冲突 | 床位已被占用 | 检查床位分配逻辑，重新分配床位 |
| 权限验证失败 | 系统间认证失败 | 检查接口认证配置 |

### 异常处理流程
```
接收错误ACK → 记录错误日志 → 分析错误原因 → 人工干预 → 重新发送
```
